Using git over SSH in Cuba or drink your favorite poison

Published April 23, 2020

Using Git over the HTTPS protocol has the shortcoming, at least on my usual Linux setup interacting with GitHub, of asking user name and password for every modification on the remote branch. The solution for that is using git over SSH. You upload your SSH public key to GitHub, add the remote branch with an URL like git@github.com:user/repository.git, and after that the communication is smoother. I have the suspicion that is even more secure.

But in Cuba you have two ways of connecting to the internet:

  • Work place or school: implies you’re always behind an HTTP proxy, therefore SSH is blocked by HTTP proxy definition. Also the network administrators block entire sections of the Word Wide Web like services that can be a source of privacy, information, political activism or revenue for the users. I’ve seen them block all the webmail providers of the planet, using blacklists they download from the WWW.
  • WiFi, home ADSL and mobile data: the state monopoly of telecommunications also blocks services with the same objective as above, but not so aggressively, however SSH sometimes is blocked. The aggressiveness is kept in the prices of those services: having an average salary of 42 dollars per month, Cubans pay 1 dollar for one hour of WiFi, 15 dollars for 30 hours of home ADSL and 7 dollars for 600MB of mobile data.

As usual Internet situation in Cuba is summarized with «Drink your favorite poison». Then you need to develop an imperturbable calm, because at some point you’ll listen government officials congratulate themselves, repeatedly, for increasing Internet access in the country under harsh economical sanctions by USA. They never mention that the so called harsh sanctions have exceptions for telecommunications equipment. Even in the Obama era they were offered free infrastructure, later rejected because Internet access without strict control could lead to a regime change.

In general if you want to use SSH in Cuba you need to rely on Tor or a free VPN, because it’s impossible to make online payments from within the country (maybe some friend abroad could help you). Tor provides something marvelous for the nerds inside the island: a SOCKS proxy with Internet access. With that you can tunnel SSH using a simple configuration:

Create a file ~/.ssh/config with the following content:

Host github.com
	User your-github-user
	ProxyCommand nc -x localhost:9150 %h %p

For getting that working you need the netcat program installed (called nc from command line), and having the SOCKS proxy server that comes with Tor up and connected to the Internet. Fortunately Tor works under a variety of censorship techniques like an HTTP proxy.
